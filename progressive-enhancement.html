Progressive Enhancement

What is it?

Basically Progressive Enhancement is a set of principles:

  1. Your HTML should be semantically meaningful
  2. Your HTML should be independent of your CSS
  3. Your CSS should include no new information
  4. Your JavaScript should only add extra functionality

Independent CSS

Your HTML should be independent of your CSS

This is the principle that you not modify the structure of your HTML to accommodate your styling.

Adding classes and identifiers is alright but ideally you wouldn't add any new elements to your code unless they convey meaning on their own.

No New Information in CSS

Your CSS should include no new information

You should have no information in your document that is conveyed only in CSS.

It's okay to do something like add color coding to a document but document should communicate the same information without it.

This is largely for accessibility (screen readers don't understand color coding).

JavaScript

Your JavaScript should only add extra functionality

This is the biggest difference between a website and a webapp.

Progressive Enhancement only really applies to websites. If you're building something that is interactive by nature then JavaScript will be critical.

If you're primarily presenting information though you should do your best to make it accessible without JavaScript.
