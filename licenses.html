The Types of Software Licenses

Overview

There are five main kinds of software licenses.

  • Proprietary
  • Public Domain
  • Permissive
  • Weak "Copyleft"
  • Strong "Copyleft"

Once you pick a category read up on the specific licenses it includes.

Proprietary

All Rights Reserved

The default under copyright law in most jurisdictions.

Pros

  • If it's a side project you own it
  • Easy to commercialize

Cons

  • Your employer probably owns it
  • Closed Source

Public Domain

I release this into the public domain

If you don't care what people do with your code you can release it into the Public Domain.

Pros

  • Simple

Cons

  • You no longer own it

Public Domain (examples)

Github provides the following public domain equivalent licenses:

  • Creative Commons Zero v1.0 Universal
  • The Unlicense

Ways they differ from saying "I release this into the public domain":

  • Works internationally
  • Warranty free disclaimer
  • Liability disclaimer

Permissive

These licenses are all pretty much "do whatever you like with this code".

Pros

  • You can commercialize it
  • Can be mixed into almost any project safely
  • You retain ownership

Cons

  • Someone else can commercialize it

Permissive (examples)

Github provides the following permissive licenses:

  • Apache License 2.0
  • MIT License
  • BSD 2-Clause "Simplified" License
  • BSD 3-Clause "New" or "Revised" License
  • Boost Software License 1.0

Copyleft

These licenses require releasing changes under the same license.

  • Comes in "weak" and "strong" varieties
  • The most restrive open source licenses
  • Intended as the opposite of copyright
  • Use copyright itself to accomplish this goal
  • Sometimes called "share alike"
  • Sometimes called "viral"

Weak Copyleft

Weak copyleft licenses apply only to the changes made directly to a project.

Pros

  • All changes must be open sourced
  • You retain ownership
  • Difficult to commercialize

Cons

  • Difficult to commercialize
  • Can't hide "secret sauce"

Weak Copyleft (examples)

Github provides the following weak copyleft licenses:

  • Eclipse Public License 2.0
  • GNU Lesser General Public License v2.1
  • Mozilla Public License 2.0

Strong Copyleft

Strong copyleft licenses apply to all changes made to a project and anything that uses it as a dependency.

Pros

  • All changes must be open sourced
  • Dependents must be open sourced
  • You retain ownership
  • Difficult to commercialize

Cons

  • Difficult to commercialize
  • Can't hide "secret sauce"
  • Scares lawyers

Strong Copyleft (examples)

Github provides the following strong copyleft licenses:

  • GNU General Public License v3.0
  • GNU Affero General Public License v3.0
  • GNU General Public License v2.0

Creative Commons

These are related are not recommended for software use (aside from CC0). They're a good fit for assets you create (or want to use) in your projects though.

They come in various combinations that include one or more of:

  • Attribution (aka "BY")
  • Share Alike (aka "SA")
  • Non Commercial (aka "NC")
  • No Derivatives (aka "ND")
