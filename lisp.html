Lisp

What is Lisp?

Lisp (short for "list processing") is actually a family of programming languages and not a single programming language. The first version of Lisp is from 1958.

Lisp is notable for its early use of:

  • Garbage collection
  • Linked lists
  • First class and higher order functions
  • Macros

Dialects

Specific languages in the Lisp family are usually called "dialects".

Notable dialects are:

  • Common Lisp
  • Clojure
  • Scheme
  • Racket (an offshoot of Scheme)
  • Emacs Lisp

Racket

Racket is a teaching focused offshoot of Scheme, the examples in this presentation will be written in Racket but the concepts are broadly applicable.

Aside from the syntax Racket should be a fairly familiar language to you. It's dynamically typed and uses functions as first class objects.

Where it will feel different is that Racket (as with most Lisps) doesn't have iterative constructs but instead relies on recursion.

Lists

The basic building block of Lisp programs is the List.

The thing about Lisp code is that it's actually a datastructure. Any Lisp program is a series of lists that are executed.

A typical function call looks like: (function-name arg0 arg1)

One way to represent something as data is to use the list function: 

(list "dog" "cat" "rabbit")

Syntax

The biggest adjustment when learning to write/read Lisp is the syntax itself, or more the lack of syntax/abundance of parenthesis.

Here is a pretty standard greeting program: 

          
(define (greet name)
  (displayln (string-append "Hello, " name "!")))

(greet "Ada") ; Prints "Hello, Ada"

Structure

Because Lisp is highly structured it's easy to reason about as a syntax tree.

In many ways this makes it resemble a markup language. 

        
(html
  (head
    (title "Lisp"))
  (body
    (section
      (h1 "Lisp"))))

Macros

Because Lisp is so datastructure oriented it's historically been much easier to write macros in it than other languges.

Macros are basically functions that get to control when they evaluate their arguments.

For example, in a language that has macros it's common for something like if to be implemented as a macro that generates a switch statement.
